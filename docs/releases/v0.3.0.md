# v0.3.0 release notes

v0.3.0 adds Local Search & Source Memory to the generated household admin workflow.

## What changed

- Added a deterministic local SQLite index for sources, cards, actions, events, and risks.
- Added a search CLI with keyword, item type, category, owner, status, privacy tier, and date-range filters.
- Added source-traceable results that show the originating note, generated output, date, and privacy tier.
- Added literal UTF-8 substring search across English, Chinese, and Norwegian synthetic content.

## New outputs

- `memory_index.md` provides a readable overview, quick links, and tables of sources, actions, and events.
- `search_index.sqlite` stores the local searchable records under ignored `output/`.

The five v0.2.0 household admin Markdown outputs remain unchanged and continue to be generated.

## Search examples

```powershell
.\.venv\Scripts\python.exe -m evergreen_memory_lite.search --memory output\household_admin --query kvittering
.\.venv\Scripts\python.exe -m evergreen_memory_lite.search --memory output\household_admin --query 收纳箱
.\.venv\Scripts\python.exe -m evergreen_memory_lite.search --memory output\household_admin --item-type action
.\.venv\Scripts\python.exe -m evergreen_memory_lite.search --memory output\household_admin --privacy-tier P0
```

## Boundaries

- Synthetic P0 examples remain the supported default; there is no real-data support.
- Source files remain unchanged.
- Search is literal and rule-based, with no translation or semantic ranking.
- No LLM, RAG, vector database, embeddings, cloud service, server, web UI, bot, or background process is included.

## For maintainers

The v0.3.0 tag points to the final implementation and release-documentation commit after tests, safety scans, and CI pass.
