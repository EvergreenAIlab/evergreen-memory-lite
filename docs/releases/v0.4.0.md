# v0.4.0 release notes

v0.4.0 adds Document Intake + Simple Local Launcher.

## What changed

- Added `.txt`, `.md`, `.docx`, and digital `.pdf` intake.
- Added extraction previews under `output/latest/extracted/`.
- Added `extraction_report.md`.
- Added `skipped_files.md`.
- Added static local `dashboard.html`.
- Added a small Windows-first local launcher.
- Added root-level batch entry points.

## Supported inputs

- `.txt`
- `.md`
- `.docx`
- digital `.pdf` with selectable text

## Unsupported inputs

- old `.doc`
- scanned PDF
- image OCR
- encrypted PDF
- email/cloud/bot/background ingestion

## User workflow

1. Put supported files in `inbox/`.
2. Double-click `Run Evergreen Memory Lite.bat`.
3. Click `Run inbox`.
4. Open the generated dashboard.

## Boundaries

- No LLM.
- No RAG.
- No vector database.
- No embeddings.
- No cloud.
- No OCR.
- No real sensitive data support.
- Source files remain unchanged.

## For maintainers

This release expands the input layer but keeps the system local, conservative, and source-traceable.
